# Сборка и публикация проекта

Сборка для Windows. В каталоге с проектом выполнить:
`$ dotnet publish -c Release -r win-x64`

Сборка для Linux. В каталоге с проектом выполнить:
`$ dotnet publish -c Release -r linux-x64`

# Запуск проекта

Находясь в каталоге с собранным проектом запустить для Windows:

`$ EventManagementServices.exe`

для Linux:

`$ ./EventManagementServices`

В Development доступен Swagger. 

# HTTP API

## Получение списка всех событий

Хендлер: `GET /events`.

Возвращает список всех событий.

Возможные коды ответа:

200 - успешное получение списка событий.

## Получение события по id

Хендлер: `GET /events/{id}`.

Возвращает событие по указанному идентификатору.

Возможные коды ответа:

200 - событие найдено.
404 - событие не найдено.

## Создание события

Хендлер: `POST /events`.

Создаёт новое событие.

Возможные коды ответа:

201 - событие успешно создано.
400 - ошибка валидации входных данных.

## Обновление события

Хендлер: `PUT /events/{id}`.

Обновляет существующее событие.

Возможные коды ответа:

204 - событие успешно обновлено.
404 - событие не найдено.
400 - ошибка валидации.

## Удаление события

Хендлер: `DELETE /events/{id}`.

Удаляет событие по идентификатору.

Возможные коды ответа:

200 - событие успешно удалено.
404 - событие не найдено.
